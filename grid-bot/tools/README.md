# Диагностические скрипты

Не используются в продакшене. Помогли в разработке найти и починить специфику Bybit API.

Все скрипты надо запускать из корня проекта `grid-bot/`:

```cmd
cd e:\AI\AI_folder\grid-bot
.venv\Scripts\python.exe tools\diag_balance.py
```

## Что какой делает

- **diag_balance.py** — диагностика чтения баланса с Bybit Demo. Использовался когда выяснилось что `fetch_balance()` через ccxt не работает на demo, и пришлось делать прямой вызов `/v5/account/wallet-balance`.
- **diag_order.py** — тестовое размещение одного ордера. Поймали bug с `retCode` приходящим строкой "0" вместо int 0.
- **diag_quick.py** — быстрая проверка какие пары торгуются на Bybit (MATIC уже не отдаёт свечей).
- **diag_backtest.py** — бэктест на 7/30/90 дней по BTC/USDT.
- **diag_backtest_compare.py** — сравнительный бэктест по 10 монетам.

Можно удалить целиком если не нужны. Они не подключены ни к серверу, ни к UI.
